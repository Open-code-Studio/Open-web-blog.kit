扩展系统 v1.0 — 开发者指南

✨ 扩展系统开发者指南

本页面既是一个可运行的扩展示例，也是一份完整的开发文档。阅读本文档后，你将能够为博客创建任意类型的扩展：外观主题、新页面、交互功能。

🚀 快速开始（5 分钟上手）

创建一个最简扩展只需 3 个文件：

步骤 1 — 创建目录和配置文件
    
    expand/my-ext/config.json:
    {
      "name": "My Extension",
      "version": "1.0.0",
      "type": "theme",
      "css": ["style.css"]
    }
步骤 2 — 编写样式
    
    expand/my-ext/style.css:
    :root { --md-sys-color-primary: #FF6B6B; }
步骤 3 — 启用扩展
    
    编辑 expand/config.json，添加：
    "my-ext": { "enabled": true }

刷新页面即可看到效果。就是这么简单。

📁 完整目录结构

Blog/                          ← 站点根目录
├── expand/                    ← 扩展根目录
│   ├── config.json            ← 【总控】启用/禁用所有扩展
│   ├── expand-test/           ← 示例扩展（你正在看的）
│   │   ├── config.json        ← 【定义】扩展的名称/文件/页面
│   │   ├── expand.css         ← 外观样式（可多文件）
│   │   └── expand.html        ← 扩展页面（可多页面）
│   ├── my-theme/              ← 你的扩展目录
│   │   ├── config.json
│   │   ├── dark.css
│   │   └── home.html
│   └── my-widget/             ← 另一个扩展
│       ├── config.json
│       ├── widget.js
│       └── widget.html
├── main/                      ← 站点核心代码（勿修改）
│   ├── js/extensions.js       ← 扩展加载器
│   ├── css/                   ← 站点样式
│   └── ...
└── page/                      ← 博客文章 (.md)

重要：每个扩展有独立的子目录。扩展之间不会互相干扰。扩展加载顺序由 expand/config.json 中的键顺序决定。

📋 配置文件 Schema（供 AI 解析）

3.1 总控文件：expand/config.json

{
    "extensions": {
      "扩展目录名": {
        "enabled": true | false
      }
    }
  }
示例：
  {
    "extensions": {
      "my-theme":   { "enabled": true },
      "my-widget":  { "enabled": false }
    }
  }

3.2 扩展定义文件：expand/{目录名}/config.json

{
    "name":         "扩展显示名称",
    "description":  "功能描述",
    "version":      "1.0.0",
    "type":         "theme" | "page" | "both",
    "css": [
      "style.css",
      "dark-theme.css"
    ],
    "scripts": [
      "widget.js"
    ],
    "pages": [
      {
        "file":      "guide.html",
        "title":     "完整页面标题",
        "navTitle":  "导航简称",
        "icon":      "code",
        "tags":      ["教程", "扩展"],
        "date":      "2026-06-27",
        "excerpt":   "页面摘要描述"
      },
      {
        "file":      "about.html",
        "title":     "关于此扩展",
        "navTitle":  "关于",
        "icon":      "book"
      }
    ]
  }
字段说明：
  css[]         — 文件路径相对于扩展目录。支持任意数量的 CSS 文件。
  scripts[]     — 文件路径相对于扩展目录。脚本以 defer 方式加载。
  pages[]       — 每个页面对象定义一个可导航的 HTML 页面。
  pages[].file  — 页面路由为 /ext/{扩展目录名}/{此文件名}
  pages[].icon  — 可选值见下方「可用图标」
  pages[].tags  — 首页的分类卡片和标签筛选会自动包含这些标签

如果只做外观扩展，只需填写 css 字段。如果只做页面扩展，只需填写 pages 字段。两者都填就是 "type": "both"。

🎨 外观扩展开发

外观扩展通过加载 CSS 文件来修改站点的视觉风格。CSS 会以 <link> 标签注入到页面 <head> 中。

修改主题色

覆盖 --md-sys-color-primary 等 CSS 变量

自定义字体

加载 Google Fonts 并设置 body 的 font-family

调整布局

修改 .blog-hero、.post-card 等布局类

添加动画

定义 @keyframes 并应用到页面元素

示例：将一个 CSS 文件改为深色主题
  expand/my-dark-theme/dark.css:

  /* 覆盖表面颜色为深色 */
  :root, [data-theme="light"] {
    --md-sys-color-surface: #1a1a2e;
    --md-sys-color-on-surface: #e0e0e0;
    --md-sys-color-surface-container: #16213e;
    --md-sys-color-surface-container-low: #0f3460;
    --md-sys-color-primary: #e94560;
    --md-sys-color-on-primary: #ffffff;
  }

  /* 修改卡片样式 */
  .post-card {
    border-radius: 8px;
    box-shadow: 0 4px 20px rgba(233,69,96,0.2);
  }

提示：站点使用 Material Design 3 设计令牌（CSS 自定义属性）。你可以在浏览器的 DevTools 中查看 :root 和 [data-theme="dark"] 下的所有可用变量。所有变量名以 --md-sys-color- 开头。

📄 页面扩展开发

页面扩展添加全新可导航的 HTML 页面。内容通过 AJAX 加载到文章视图容器中（#docBody），站点导航栏和侧边栏自动包含你的页面链接。

页面 HTML 规范
  你的 HTML 不需要 <html>、<head>、<body> 标签。
  直接写内容即可，它会被注入到 #docBody 容器内。

  <div class="ext-page-container">
    <h2>我的扩展页面</h2>
    <p>页面内容...</p>

    <div class="ext-feature-grid">
      <div class="ext-feature-card">
        <h4>卡片标题</h4>
        <p>卡片内容</p>
      </div>
    </div>

    <div class="ext-code-block">
      代码块内容
    </div>

    <span class="ext-badge">标签</span>
  </div>

可用的预定义 CSS 类：

类名 用途
.ext-page-container 扩展页面外层容器（推荐使用）
.ext-feature-grid 响应式卡片网格（2-4 列自动适配）
.ext-feature-card 带悬停效果的 MD3 风格卡片
.ext-code-block 等宽字体代码展示块
.ext-badge 胶囊形状态/版本标签

你也可以使用站点自带的任何 MD3 类，如 .post-card、.post-card__tag、.empty-state 等。

⚡ 脚本扩展开发

通过 scripts 字段加载自定义的 JavaScript。脚本以 defer 方式加载，不阻塞页面渲染。

示例：添加一个浮动按钮
  expand/my-widget/widget.js:

  (function() {
    'use strict';

    // 创建浮动按钮
    const btn = document.createElement('button');
    btn.textContent = '💬';
    btn.style.cssText =
      'position:fixed;bottom:100px;right:24px;' +
      'width:56px;height:56px;border-radius:50%;' +
      'border:none;font-size:24px;cursor:pointer;' +
      'z-index:50;box-shadow:var(--md-sys-elevation-3);' +
      'background:var(--md-sys-color-primary-container);' +
      'color:var(--md-sys-color-on-primary-container);';

    btn.onclick = () => alert('Hello from extension!');
    document.body.appendChild(btn);
  })();

全局对象参考：扩展系统在 window.__EXTENSIONS 上暴露了以下数据：

window.__EXTENSIONS = {
    loaded: [],      // 已加载的扩展列表 [{ id, config }]
    pages:  {},      // 已注册的扩展页面 { 'ext-id/file': { route, title, ... } }
    styles: []       // 已注入的 CSS <link> 元素列表
  };

脚本可以使用站点已加载的任何库：Marked.js（Markdown 渲染）、Highlight.js（代码高亮）。

🎯 可用图标

在 pages[].icon 字段中使用以下值：

"code"    → 代码图标 </>
"book"    → 书本/文章图标
"puzzle"  → 拼图（扩展默认图标）
"home"    → 首页图标
"globe"   → 地球/网络图标

🤖 AI 开发指南（供 VibeCoding/Copilot/Cursor 等）

如果你是一个 AI 编程助手（如 VibeCoding），请遵循以下规范来为用户创建扩展：

## AI Agent Instructions for Extension Development

### 1. When user asks to create an extension:
  - Create a new directory under expand/ with a short kebab-case name
  - Create config.json following the schema in Section 3.2
  - Create the required CSS/HTML/JS files
  - Update expand/config.json to enable it

### 2. config.json minimum fields:
  { "name": "...", "version": "1.0.0", "type": "theme|page|both", ... }

### 3. CSS guidelines:
  - Use MD3 CSS custom properties (--md-sys-color-*) for theming
  - Prefix custom classes with ext- to avoid conflicts
  - Only include necessary overrides, don't duplicate site styles
  - File path in css[] is relative to the extension directory

### 4. HTML page guidelines:
  - Do NOT include <html>, <head>, or <body> tags
  - Wrap content in <div class="ext-page-container">
  - Use semantic HTML: h2, h3, p, ul, ol, code, table
  - Available icons: "code", "book", "puzzle", "home", "globe"

### 5. JS guidelines:
  - Wrap in IIFE: (function() { 'use strict'; ... })();
  - Access extension system via window.__EXTENSIONS
  - Available libraries: marked, hljs (may not be on all pages)
  - Be defensive: check element existence before manipulation

### 6. After creating files, update expand/config.json:
  {
    "extensions": {
      "existing-ext": { "enabled": true },
      "new-ext":      { "enabled": true }   ← add this line
    }
  }

### 7. File naming conventions:
  - config.json (required, lowercase)
  - CSS files: lowercase, hyphenated, .css extension
  - HTML files: lowercase, hyphenated, .html extension
  - JS files: lowercase, hyphenated, .js extension

🔧 常见问题排查

扩展没生效？

① 检查 expand/config.json 中 enabled 是否为 true
② 打开浏览器控制台查看 [Extensions] 日志
③ 确认目录名与 config 中的键完全一致
④ 确认扩展的 config.json 是合法的 JSON

CSS 不生效？

① 检查选择器优先级，可能需要 !important
② 确认 CSS 文件路径没有拼写错误
③ 在 DevTools 的 Elements 面板查看是否加载
④ 确认修改的是正确的 CSS 变量名

页面 404？

① 确认 HTML 文件存在于扩展目录下
② 确认文件名与 config 中完全一致
③ 路由格式：page.html#/ext/{扩展目录名}/{文件名}
④ 检查浏览器 Network 面板

脚本不执行？

① 检查控制台是否有 JS 错误
② 确认 scripts 数组中的文件名正确
③ 脚本以 defer 加载，检查 DOM 就绪状态
④ 检查是否依赖了未加载的库

💡 进阶技巧

  1. CSS 层叠顺序：后加载的扩展 CSS 会覆盖先加载的。扩展 CSS 在站点核心样式之后加载，天然具有更高优先级。
  2. 深色/浅色主题：使用 [data-theme="dark"] 选择器来分别定义深浅色主题的样式。
  3. 响应式设计：在 CSS 中使用 @media (max-width: 768px) 适配移动端。
  4. 页面中加载图片：图片路径使用相对于站点根目录的路径，如 /expand/my-ext/logo.png。
  5. 多个 CSS 文件：如果想模块化管理样式，可以在 css 数组中列出多个文件，按顺序加载。
  6. 调试模式：打开浏览器控制台，以 [Extensions] 开头的日志会显示扩展加载状态。
  7. 禁用扩展：无需删除文件，只需在 expand/config.json 中将 enabled 改为 false 即可。

💡 现在就开始：复制 expand-test/ 目录，改名为你的扩展名，修改 config.json 中的内容，然后刷新页面即见效果。

🤝 兼容性：扩展系统使用标准 Web API（fetch、DOM、CSS），无需安装任何工具或框架。所有现代浏览器均支持。
